1. Executive Summary

MailXpose is a cloud-based cybersecurity platform built to detect phishing, malware distribution attempts, and social engineering threats concealed within email communications. The system provides organizations and individuals with an intelligent, easy-to-use tool that requires no IT expertise to operate.

At its core, MailXpose uses a proprietary multi-factor threat scoring engine that evaluates each email against a comprehensive set of forensic parameters — assigning a Risk Score from 0 to 100 and classifying emails as Safe, Suspicious, or Malicious. The platform is accessible through a full-featured web dashboard and a Google Chrome Extension that integrates directly with Gmail for real-time scanning.

✅ Live in Production
MailXpose v1.0 is fully deployed and operational. The Chrome Extension is packaged and ready for Chrome Web Store submission.

2. Problem Statement & Objectives

2.1 Problem Statement

Email phishing remains the most widespread cyberattack vector globally, accounting for the majority of documented data breaches. The core challenge is that modern phishing emails are highly sophisticated — they convincingly mimic legitimate brands, government agencies, and financial institutions, making them difficult to identify with the naked eye.

Most available security solutions are either expensive enterprise-grade tools requiring dedicated IT teams, or passive spam filters that cannot detect targeted, context-aware attacks. Individual users — freelancers, small business owners, students — are left largely unprotected. MailXpose addresses this gap by providing an accessible, self-service forensic email screening tool that any user can operate without technical knowledge.

2.2 Project Objectives

  • Build an automated email threat detection system usable by non-technical users.
  • Develop a Gmail-integrated Chrome Extension for real-time one-click scanning.
  • Generate detailed forensic reports with clear threat indicators and actionable recommendations.
  • Implement a tiered subscription model to monetize premium features.
  • Deploy as a scalable, cloud-hosted SaaS product accessible from any device.
  • Ensure strong data security and user privacy compliance throughout.

3.2 Chrome Browser Extension

The MailXpose Chrome Extension integrates directly into the Gmail web interface. Upon installation and account linking, users see a "Scan with MailXpose" button within each email. The extension reads the current email and sends it for analysis, then displays a real-time risk overlay within Gmail — without navigating away from the inbox.

  • Supports both manual and automatic scan modes

5. System Architecture

5.1 Architecture Overview

MailXpose follows a three-tier client–server–database architecture. The frontend (web dashboard and Chrome Extension) communicates with a centralized backend API over secure HTTPS. The backend applies business logic — including authentication, subscription enforcement, and forensic analysis — before persisting results to the cloud database.

5.2 Data Flow

  1. User Login: Credentials verified → secure session token issued to the client.
  2. Email Submission: Email data (subject, sender, body, authentication headers) submitted for analysis.
  3. Forensic Processing: The analysis engine evaluates all threat parameters and computes a risk score.
  4. Report Generation: A structured forensic report is saved and returned to the client instantly.
  5. Dashboard View: User accesses full scan history, statistics, and individual report details.
  6. Extension Sync: The Chrome Extension fetches live data and displays results within Gmail.
ℹ️ Data Storage
The system stores email metadata (subject, sender address, risk scores, threat indicators) necessary for generating reports and analytics. Raw email body content is processed in-memory during analysis and is only retained if the user has opted into detailed report storage.

6. Email Threat Detection Engine

The detection engine is the core intellectual component of MailXpose. It performs a simultaneous, multi-dimensional forensic evaluation of each email and produces a deterministic risk assessment. The engine runs entirely server-side and is invisible to the end user — results are returned within milliseconds.

6.1 Analysis Parameters

Parameter What Is Checked Impact
Email Header Authentication SPF & DKIM record verification against sender domain High
Sender Domain Reputation Whether the sender's domain is known, trusted, or suspicious Medium
Subject Line Analysis Detection of urgency-inducing or pressure language Medium
Keyword Pattern Matching Presence of phishing and malicious vocabulary in body text High
Embedded Link Analysis Number of links, detection of shortened/obfuscated URLs Medium
Attachment Risk Screening Detection of dangerous file types (executables, macros, scripts) Critical
Formatting Anomalies Structural irregularities common in auto-generated phishing emails Low

6.2 Risk Classification

All parameter scores are aggregated into a final Risk Score, clamped within a 0–100 range. Verified security indicators (valid SPF, DKIM, trusted domain) reduce the overall score. The score maps to one of three risk classifications:

🟢  SAFE  — Score 0 to 39No immediate threats detected
🟡  SUSPICIOUS  — Score 40 to 69Exercise caution, verify sender
🔴  MALICIOUS  — Score 70 to 100Do not interact. Report immediately

6.3 Threat Categories

Threat Category Description Severity
Email Spoofing Sender identity falsified; authentication headers don't match High
Authentication Failure Missing or invalid email security signatures Medium
Social Engineering Urgency tactics, fear-based language to pressure the recipient Medium
Phishing Language Vocabulary patterns associated with credential harvesting attacks Medium
Malware Distribution Language or attachments associated with malware campaigns High
Suspicious URL Embedded links using shortened or obfuscated URLs High
Dangerous Attachment Executable, script, or macro-enabled file types detected Critical

8. Security & Privacy

8.1 Security Measures

  • All user passwords are stored using strong cryptographic hashing — never in plaintext.
  • Session management is handled through short-lived, signed authentication tokens.
  • All data in transit is encrypted using HTTPS / TLS.
  • HTTP security headers enforced to protect against common web vulnerabilities (XSS, clickjacking, CSRF).
  • Payment processing is handled entirely by a PCI-compliant third-party gateway; MailXpose never stores card data.

8.2 Privacy Policy

MailXpose processes email content solely for the purpose of threat analysis. Email body data is not shared with third parties, not used for training or advertising, and is deleted from processing memory after analysis. Users may request deletion of their stored report data at any time. A comprehensive Privacy Policy is published on the platform.

⚠️ Extension Privacy Note
The Chrome Extension requests permission to read the currently active Gmail tab only when the user initiates a scan. The extension does not read emails passively in the background unless real-time scanning is explicitly enabled by the user.

10. Limitations & Future Scope

10.1 Current Limitations

  • The current detection engine uses a rule-based approach. Deep learning / NLP-based models are planned for a future release.
  • The Chrome Extension currently supports Gmail only. Support for Outlook and Yahoo Mail is planned.
  • Attachment analysis is limited to file-type screening. A sandboxed execution environment for deeper file analysis is a future goal.
  • Mobile access is currently browser-based. A dedicated native mobile app is under consideration.

10.2 Future Scope

  • Machine Learning Integration: Train classification models on labeled phishing datasets for AI-driven detection with higher accuracy and fewer false positives.
  • Multi-client Extension Support: Expand the browser extension to support Outlook Web, Yahoo Mail, and other providers.
  • Threat Intelligence Feeds: Integrate with open-source threat databases to cross-reference known phishing domains and malicious IPs in real time.
  • Mobile Application: Develop a companion mobile app (Android & iOS) with push alerts for immediate threat notifications.
  • Enterprise SSO: Support SAML 2.0 and OAuth 2.0 for corporate single sign-on deployments.
  • Two-Factor Authentication: Add TOTP-based 2FA for enhanced account security.
  • Team / Organization Accounts: Allow multiple users under a single organization with centralized admin controls.
